# Guía de Contribución

Gracias por tu interés en contribuir a este proyecto 

## Flujo de trabajo

1. Crear una rama nueva desde `main`:
   - `feature/nombre-feature`
   - `fix/nombre-arreglo`
2. Realizar commits pequeños y con mensajes descriptivos.
3. Abrir un Pull Request hacia la rama `main`.
4. Esperar la revisión al menos de un miembro del equipo.

## Estándares de código

- Seguir PEP8 en todo el código Python.
- Mantener los notebooks limpios (celdas ejecutadas en orden).
- Comentar las partes del código que no sean evidentes.

## Tests

Antes de realizar un Pull Request:

- Ejecutar los tests de la carpeta `tests/` (si existen).
- Asegurarse de que el proyecto se ejecuta sin errores.

## Issues

- Usar etiquetas: `bug`, `enhancement`, `documentation`, etc.
- Explicar claramente el problema y cómo reproducirlo.
